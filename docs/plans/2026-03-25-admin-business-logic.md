# 管理端业务逻辑设计文档

> 创建时间：2026-03-25

## 一、定位

管理端不是单纯的页面集合，而是直播业务中台。

它负责三类事情：
- 维护基础主数据
- 配置直播业务规则
- 管理直播运行与回放结果

页面只是业务逻辑的承载层，开发顺序必须先理清数据关系，再落页面。

## 二、直播业务的正确划分

### 1. 直播方式
- 真实直播
- 伪直播

说明：
- 公开课不是单独直播方式，本质仍然是真实直播
- 公开课与普通真实直播的区别，在于学生进入方式和聊天规则

### 2. 学生进入方式
- 接口进入
- 白名单进入
- 分享链接进入

说明：
- 接口进入：学生从教务/学习平台进入直播列表并点击上课
- 白名单进入：通过名单控制可进入学生范围
- 分享链接进入：公开课场景，任何人可进入

### 3. 回放属性
- 每场直播天然都会生成回放
- 管理端需要配置的是：是否允许学生观看回放
- 回放不是直播方式，也不是单独业务入口

## 三、三端入口链路

### 1. 老师端

老师进入方式有三条：

1. 成教平台路径
- 成教平台教师端进入课程
- 进入学习平台直播列表
- 点击具体课程
- 直接唤起客户端并开播

2. 客户端房间号登录
- 输入房间号 + 密码
- 直接进入直播间开播

3. 客户端机构账户登录
- 输入机构名称
- 输入机构账号密码
- 进入直播列表
- 选择课程进入直播间

结论：
- 老师没有独立注册体系
- 老师数据来自机构侧或业务平台同步
- 管理端必须维护机构、教师、房间号、密码、教师与课程绑定关系

### 2. 学生端

学生进入方式有两条：

1. 教务平台进入
- 登录教务/学习平台
- 查看直播列表
- 点击直接上课

2. 分享链接进入
- 打开公开课分享页
- 点击进入观看

结论：
- 学生没有独立登录页
- 学生数据来自教务平台、课程关系或白名单
- 管理端必须维护学生、课程、班级、白名单和分享规则

## 四、管理端数据分层

### 第一层：基础主数据

1. 机构
- 机构名称
- 机构状态
- 区域
- 标签
- AppKey / AppSecret
- 对接域名 / 接口路径

2. 教师
- 教师姓名
- 工号/账号
- 归属机构
- 可授课课程
- 是否有直播权限

3. 学生
- 学生姓名
- 学号/用户ID
- 归属机构
- 所属班级
- 所属课程
- 是否有观看权限

4. 课程
- 课程名称
- 课程编号
- 归属机构
- 对应教师
- 对应班级

5. 班级
- 班级名称
- 归属机构
- 学生列表

### 第二层：直播配置数据

1. 直播间
- 房间名称
- 房间 UUID
- 房间号
- 房间密码
- 归属机构
- 对应课程
- 主讲教师
- 助教

2. 直播规则
- 直播方式：真实直播 / 伪直播
- 学生进入方式：接口 / 白名单 / 分享链接
- 聊天规则：允许聊天 / 全员禁言
- 摄像头规则：是否显示老师摄像头区
- 回放权限：是否允许学生观看回放

3. 场次配置
- 场次开始时间
- 场次结束时间
- 课节时长
- 状态：未开始 / 直播中 / 已结束 / 异常结束

### 第三层：直播运行数据

1. 直播列表
- 直播实例主表
- 展示直播运行结果与管理入口

2. 进出教室记录
- 老师进入/退出
- 学生进入/退出
- 首次进入 / 最后退出
- 累计观看时长
- 中途离开次数
- 迟到 / 早退

3. 授课统计方式
- 用于定义或展示时长、考勤、统计口径

4. 直播回放列表
- 回放片段
- 片段状态
- 是否可见
- 补录视频

5. 直播监课
- 正在直播的场次
- 在线人数
- 聊天状态
- 禁言状态
- 强制关闭

## 五、实体关系

基础关系：
- 一个机构有多个教师
- 一个机构有多个学生
- 一个机构有多个课程
- 一个课程关联一个或多个班级
- 一个班级关联多个学生

直播关系：
- 一个直播间归属于一个机构
- 一个直播间绑定一个课程
- 一个直播间绑定一个主讲教师，可选绑定助教
- 一个直播间下可以有多个场次
- 一个场次会生成一条直播列表记录
- 一个场次对应一组进出教室记录
- 一个场次对应一组回放片段

## 六、管理端正确菜单顺序

不是先做直播列表，而是按业务依赖顺序设计：

1. 基础数据
- 机构管理
- 教师管理
- 学生管理
- 课程管理
- 班级管理

2. 直播配置
- 直播间管理
- 场次创建
- 白名单配置
- 分享规则配置

3. 直播运行
- 直播列表
- 进出教室记录
- 授课统计方式
- 直播回放列表
- 直播监课

4. 辅助模块
- 机构对接配置
- 模拟直播

## 七、页面开发顺序

第一阶段：先把基础数据做全
- 机构管理
- 教师管理
- 学生管理
- 课程管理
- 班级管理

第二阶段：再做直播配置
- 直播间管理
- 场次创建
- 进入方式配置
- 聊天与回放权限配置

第三阶段：最后做运行页
- 直播列表
- 进出教室记录
- 直播回放列表
- 直播监课

## 八、当前开发约束

- 不再先从页面美化出发
- 后续管理端开发以业务数据关系为先
- 页面结构必须服从业务顺序，而不是反过来
