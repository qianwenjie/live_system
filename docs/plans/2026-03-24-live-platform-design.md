# 直播平台设计文档

> 创建时间：2026-03-24

## 一、系统概述

SaaS 形式的直播平台，服务多个机构（学校），各机构数据完全隔离。平台对外不暴露多机构性质，每个机构感知上是独立的直播系统。

---

## 二、技术栈

| 子系统 | 技术 |
|--------|------|
| 老师客户端 | Electron + Vue2（Windows/macOS） |
| 学生观看端 | Vue2 网页 |
| 平台管理后台 | Vue2 网页 |
| 后端服务 | Java Spring Boot |
| 流媒体 | 腾讯云标准直播（CSS） |
| 伪直播视频存储 | 腾讯云 VOD |
| 实时聊天 | WebSocket 自建 或 腾讯云 IM |

---

## 三、机构体系

- 平台管理员在后台新增机构，生成 AppKey + AppSecret
- 机构名称用于老师客户端登录时精确匹配
- 机构通过密钥调用直播平台 API 对接
- 对接方式：机构主动调用 API + 直播平台回调推送
- 对接配置：回调接收地址、数据推送格式

---

## 四、账号与登录体系

### 老师登录（三种路径）

1. **教务平台唤起**
   - 教务平台 → 学习平台直播列表 → 点击唤起客户端 → 携带 Token 自动登录 → 直接进入直播间

2. **客户端直接启动 - 房间号登录**
   - 输入房间号 + 密码 → 直接进入该直播间

3. **客户端直接启动 - 机构账户登录**
   - 输入机构名称（精确匹配）→ 输入机构账号密码 → 进入该账号下直播课列表 → 选择课程进入直播间

> 注：直播平台无老师账号体系，每个直播间有独立房间号 + 密码，老师账号来自业务平台

### 学生登录

- 统一从教务平台 → 学习平台直播列表 → 点击进入网页观看
- 进入直播间权限三种模式（可叠加）：
  - **白名单**：手动维护或批量导入名单
  - **系统对接**：教务平台对接，课程学生自动有权限
  - **匿名**：分享链接，任何人可进，无需登录

## 五、老师客户端功能

### 进入前
- 摄像头 + 麦克风设备检测

### 直播控制
- 屏幕共享：选择桌面或应用窗口（主画面）
- 摄像头：可开关，开启时支持虚拟背景，画中画叠加在屏幕共享上
- 麦克风：可开关，实时显示音量输出状态
- 混流后推一路 RTMP 流到腾讯云 CSS

### 直播间管理
- 查看在线学生列表
- 单人禁言 / 全体禁言
- 查看聊天消息

---

## 六、学生观看端功能

- 拉流播放（主画面屏幕共享 + 画中画摄像头）
- 文字聊天（受禁言控制）
- 无其他互动功能

---

## 七、直播类型

### 真实直播
- 老师客户端实时推流

### 伪直播
- 管理员上传视频，设定开播时间
- 到时间自动触发，腾讯云 CSS 点播转直播推流
- 学生端体验与真实直播一致，支持聊天

### 公开课
- 匿名链接进入，无需登录
- 默认全体禁言，不开放聊天

---

## 八、平台管理后台

### 机构管理
- 新增/编辑/禁用机构
- 生成 AppKey + AppSecret
- 机构名称配置
- 对接配置：回调接收地址、数据推送格式

### 直播间管理
- 创建/编辑/删除直播间，归属机构
- 配置房间号 + 密码
- 进入方式：白名单 / 系统对接 / 匿名（可叠加）
- 直播类型：真实直播 / 伪直播

### 直播场次管理
每场直播独立场次，记录：
- 课程总时长、开始/结束时间
- 老师：进入时间、离开时间、中途退出/返回记录
- 每个学生：进入时间、离开时间、是否迟到/早退、中途退出/返回记录、发言次数、累计观看时长
- 行为数据回传机构教务平台

### 直播监课
- 实时查看所有正在直播的课
- 支持强制关闭直播间

### 直播回放
- 腾讯云自动录制生成回放
- 管理员手动上传兜底
- 是否允许观看由业务端控制，每个直播间独立配置

### 学生白名单
- 按直播间维护，支持批量导入

### 聊天管理
- 全局敏感词配置
- 查看各直播间聊天记录

### 模拟直播
- 模拟老师身份进行直播测试
- 模拟学生身份观看测试

### 数据统计
- 直播间维度：在线人数、观看时长
- 机构维度：汇总数据

---

## 九、对接层 API

### 机构主动调用（AppKey + AppSecret 签名）
- 同步学生白名单
- 创建/更新直播间
- 获取直播间进入链接（学生端）
- 唤起老师客户端（携带 Token 自动登录）

### 直播平台回调给机构
- 直播开始/结束通知
- 学生行为数据回传（进出记录、发言次数、观看时长）
- 考勤数据回传（迟到/早退标记）

---

## 十、技术架构关键点

### 推流链路
```
老师客户端采集（屏幕 + 摄像头可选 + 麦克风可选）
→ 本地混流
→ 推 RTMP 到腾讯云 CSS
→ 腾讯云转码
→ 生成 HLS/FLV 拉流地址
→ 学生端拉流播放
```

### 伪直播链路
```
视频存储腾讯云 VOD
→ 后端定时任务到点触发
→ 腾讯云 CSS 点播转直播
→ 学生端拉流（体验与真实直播一致）
```

### 聊天
- WebSocket 自建或腾讯云 IM
- 消息经后端敏感词过滤后下发
- 禁言状态实时同步

### 行为数据采集
- 学生端进入/离开页面时上报事件
- 心跳机制检测中途离开（页面隐藏/断连）
- 后端汇总计算迟到/早退/累计时长

### 安全
- 拉流地址防盗链（腾讯云 Key 防盗链）
- 匿名直播间限制拉流地址有效期
- AppKey + AppSecret 签名验证
