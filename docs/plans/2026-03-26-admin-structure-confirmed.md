# 管理端结构确认文档

> 创建时间：2026-03-26
> 本文档为管理端最终确认结构，覆盖并修正 2026-03-25 两份文档中的错误设计。

## 一、核心纠正

### 1. 平台没有教师账号体系

老师不通过管理端注册或维护账号。老师进入直播的方式是：
- 房间号 + 密码登录客户端
- 机构账户登录客户端后选择场次

管理端**不维护教师主数据**，教师姓名只作为直播场次的一个字段存在。

### 2. 平台没有学生账号体系

学生不在管理端注册。学生进入直播的方式是：
- 通过教务/学习平台接口进入
- 通过分享链接进入（公开课）
- 通过白名单进入

管理端**不维护学生主数据**，白名单是直播场次的配置项，不是独立模块。

### 3. 平台不管理课程和班级

课程数据来自对接的业务平台，通过 API 同步或课程映射获取。
课程映射属于**机构对接配置**的子内容，不是独立模块。

### 4. 直播房间与场次是同一层

在对接的学习平台中，一个课程有多个直播场次。
对应到直播平台里，每个场次就是一个独立的房间。
**不存在"持久房间下挂多个场次"的层级关系。**

### 5. 摄像头不由管理端配置

摄像头由老师在直播过程中自由开启，管理端不提供摄像头规则配置项。

## 二、核心业务对象

### 1. 机构
- 机构名称、区域、状态
- AppKey、AppSecret、业务平台域名、接口路径
- 课程映射（业务平台课程标识与本平台的对应关系）

### 2. 直播场次（即直播房间）
- 房间名称、房间 UUID、房间号、房间密钥
- 归属机构
- 直播类型：真实直播 / 伪直播
- 学生进入方式：接口进入 / 白名单进入 / 分享链接进入
- 聊天规则：允许聊天 / 全员禁言
- 回放权限：是否允许学生观看回放
- 场次时间：开始时间、结束时间、课节时长
- 主讲姓名、助教姓名
- 状态：未开始 / 直播中 / 已结束 / 异常结束

### 3. 进出教室记录
- 角色（老师 / 学生）、用户ID、姓名
- 首次进入、最后退出、累计观看时长
- 是否迟到、是否早退、中途离开次数
- 进出时间轴详情

### 4. 回放片段
- 片段名称、时间范围、时长
- 生成状态、是否对学生可见
- 来源：原生回放 / 补录上传

### 5. 互动消息
- 聊天记录：时间、用户、消息内容
- 禁言记录：时间、操作人、被禁言用户、操作类型

## 三、菜单结构

```
机构与对接
  └ 机构管理
  └ 对接配置（含课程映射）

直播管理
  └ 直播列表          ← 核心主表，每条记录 = 一个场次/房间
  └ 进出教室记录
  └ 回放列表
  └ 互动消息
  └ 直播监课

辅助
  └ 模拟直播
```
